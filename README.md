# 智慧医疗挂号系统 — AI 多 Agent 挂号助手

患者端微信小程序 + Java 后端 + Python NLU 的多 Agent 智能挂号系统。LLM 做意图识别，Worker 流水线做执行，知识图谱做症状→科室推理。


qq交流群： 1081725203

## 项目结构

```
hospital
├── hospital-api-mysql          # 管理系统后端（Spring Boot）
├── hospital-vue                # 管理系统前端（Vue3 + ElementPlus）
├── patient-wx-api-mysql        # 小程序后端（Spring Boot）
├── patient-wx                  # 患者端小程序（UniApp）
├── nlu-service                 # NLU 推理服务（Python）
├── sql                         # 数据库 SQL
└── docs                        # 文档
```

```

                                                      ▼
降级 1:   rules.py 规则引擎 (10个纯函数) ──> 返回规则结果 ──> Worker 流水线
                                                      │
                                                      │ slots.department 为空
                                                      ▼
降级 2:   ① Neo4j 图谱 top1 ──无──> ② 静态映射表 ──无──> ③ 原文匹配
                                                      │
                                                      │ Python 连接拒绝 / 崩溃
                                                      ▼
降级 3:   Java @Autowired(required=false) ──> "NLU 服务不可用，请走普通挂号"
            + 手动挂号跳转卡片
                                                      │
                                                      │ Worker 只读 DB 异常
                                                      ▼
降级 4:   重试 1 次 ──仍失败──> 错误提示或普通挂号兜底卡片
```

## 多 Agent 核心设计

### Worker 分工

| Worker | 职责 | 阶段 |
| --- | --- | --- |
| Coordinator | NLU 解析、Memory 管理、意图路由、流程编排 | 入口 |
| ScheduleAgentWorker | 科室匹配→子科室→日期→医生排班→号源时段（5步顺序） | SLOT_QUERY |
| PolicyAgentWorker | 参数完整性、登录态、就诊卡、当日上限、重复挂号 | POLICY_CHECK |
| ExecutionAgentWorker | 登录/确认二次校验、pendingOrder 一致性、幂等写 | EXECUTE_APPOINTMENT |

### 写操作安全

- `requestId` 幂等键：已成功→复用结果，处理中→拦截，参数不一致→拒绝
- 写操作不自动重试，避免真实二次下单
- 确认闭环：只有 `awaitingConfirmation + pendingOrder` 且确认参数与快照一致才放行
- fail-closed：Policy 校验失败立即清除 pendingOrder，阻止进入写路径

### 会话隔离

检测到症状切换且旧科室不在新图谱中时，自动清除 `deptName/deptId/deptSubId/symptomDeptGraph`，防止跨话题污染。

## NLU 接入配置

```yaml
agent:
  multi:
    model-parser-enabled: true
    model-parser-endpoint: http://127.0.0.1:8001/infer
    model-parser-timeout-millis: 5000
    model-parser-min-confidence: 0.75
```

### 设计原则

- **Python 管理解，Java 管执行**：LLM 只出 intent + slots，写操作、排班查询、就诊卡校验全在 Java 侧
- **LLM → 规则引擎自动降级**：超时/低置信度回退 rules.py，Python 崩溃时 Java `@Autowired(required=false)` 兜底
- **科室三级兜底**：Neo4j 图谱 → 静态映射表 → 原文匹配
- **高危意图拦截**：`DANGEROUS_KEYWORDS` 黑名单 + LLM prompt 双重识别
